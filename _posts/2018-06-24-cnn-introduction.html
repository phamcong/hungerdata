Computer Vision

Human people can look at an image and then easily describe its content, recognize or detect embedded objects and their positions. Whereas this task is much more difficult for the computer since all it interpret any image as a matrix of numbers (pixels). The main goal of Computer Vision is toridge the gap between pixels and "meaning". Computer Vision is one of the fastest growing and most exciting AI disciplines in today’s academia and industry. Some problems treadted by Computer Vison:

  • Image Classification
  • Object detection: Car, perdestrican, traffic lights
  • Neural Style Transfer: Content Image + Style Images => New kind of art.

Introduction of Convolutional Neural Network

Convolutional Neural Network (CNN or ConvNet) is an artifical neural network (more detail on Wikipedia, paper, video) that is so fare been most popularly used for analyzing images. Although image analysis has been the most widespread use of CNN's, they can also be used for other data analysis or classification as well. CNN has some types of specialization for being able to pick out or detect patterns and make sense of them. This pattern dectection is what makes CNN so useful for image analysis. What is differentiates CNN from just a standard Multi-Layer Perception (MLP)? CNN has hidden layers called convolutional layers and these layers are precisely what make a CNN a CNN (😃).

Components of CNN

Convolutional Layer

Convolutional Layers are able to dectet patterns in images.
Like other hidden layer, a convolution layer recieves input, transforms it in some way and then outputs the transform input to the next layer. This transformation is a convolution operation. Each convolutional layer come with a set of filters (kernels or feature detectors) that help to detect the patterns.

What are PATTERNs? A single image contains many multiple edges, shapes, textures, objects, etc. Pattern detection filter could be edge corner, circles, square detectors.

The deeper the network goes, the more sophisticated filters become
At the start of CNN, first convolutional layers detect simple patterns (geometric filters). The deeper the network goes, the more sophisticated filters become. Later layers can detect specific objects like eyes, ears, hair. Deepest layers can detect complete objects such as full dogs, cats, lizards, birds, etc.

Padding

Downside of applying convolutional operator
  • Given a input is a matrix of size 6×66\times66×6. If we convolve it with a 3×33\times33×3 filter, we end up with a 4×44\times44×4 output (sine there are only 4×44\times44×4 possible positions for filters laying down on the input matrix). Mathematically, if we have an n×nn \times nn×n input matrix and convolve it with a f×ff\times ff×f filter, the size of output matrix is: (n−f+1)×(n−f+1)(n - f + 1) \times (n - f + 1)(n−f+1)×(n−f+1). Every time we apply a convolutional operator, the image shrinks down. So we can do this only a few times before the image start gettings really small.
  • A pixel at the middle is overlapped by a lot of 3×33\times33×3 regions, whereas, the pixels at the corner/ or the edge of input matrix is touched as used only in one of the outputs, so we are through away a lot of the information near the edge of the image.

Figure 1: Padding input matrix with additional border of p pixels.

To fix both of these two problems, we can pad the input matrix with additional border of one (some) pixel(s). For example, we pad the 6×66\times66×6 input matrix with an border of one pixel to 8×88\times88×8 matrix and then convolve it with 3×33\times33×3 filter, we will get back 6×66\times66×6 output matrix. So the size of input matrix is preserved. The pixels at the corner or the edge of the original matrix (before padding) are now more used (4 times for the ones at 4 corners).

By convention, we pad with zeros. If ppp is padding amounts (add an additional boder of ppp pixel) then the size of output matrix is now (n+2p−f+1)×(n+2p−f+1)(n+2p-f+1) \times (n+2p-f+1)(n+2p−f+1)×(n+2p−f+1).

Valid and Same convolutions
  • Valid convolutions (NO padding): (n×n)∗(f×f)(n \times n) * (f \times f)(n×n)∗(f×f) => (n−f+1)×(n−f+1)(n-f+1) \times (n-f+1)(n−f+1)×(n−f+1)
  • Same convolutions: Pad so that output size is the same as the input size. n+2p−f+1=n=>p=f−12n+2p-f+1 = n => p = \frac{f-1}{2}n+2p−f+1=n=>p=2f−1​ By convention, fff is almost always odd for two possible reasons:
    • If fff is even, we have to pad more on the left and less on the right => asymmetric.
    • If fff is odd, the filter has a central pixel. For the Computer Vision, sometimes, its nice to have a distinguisher - a pixel representing filter's position.

The way the convolutional operator is defined mathmatically is slightly different. Before doing the element-wise (dot product) and summing of the filter and the input matrix, there is one other step that need to take is take the filter and flip it on the horizontal and vertical axis. The output is calculate using the input matrix and the flipped filter. The operatation we have done so far (on the input matrix and the original filter) is sometimes called cross-correlation instead of convolution. However, by convention, in ML (or DL) this cross-correlation operator is called convolution.

Convolutions over volumnes

Convolution with one filter

Number of layers in a filter must be equal to the number of channels of the input image.
So far, the filters are applied for 2D images (gray-scale-. The convolution can be also applied for 3D images (RGB images for example). Let's take an input RGB image of size 6×66\times66×6, now we have 6×6×36\times6\times36×6×3 input matrix (333 color channels). The filter now has to have 333 layers corresponding to redredred, greengreengreen and blueblueblue channels.

Figure ..: Convolutions over volumnes - RGB image of 6x6. The filter cube is placed over the input matrix and the output is calculated by adding all multiplications of each pixel in filter layer with the pixel at corresponding layer of input matrix.

To compute the convolutional operation, we take 333 layers of the filter and place it at upper left most position and multiply them with corresponding number from the redredred, greengreengreen and blueblueblue layers. Adding up all multiplication give us the first number in the output. To compute the next output, we slide the filter cube over by one on the input matrix.

The use of filer cube allows us to detect patterns separately on each channel of RGB images. If we want to detect edges only on the redredred channel, we can then have the first layer of the filter be a edges detector while the two other layers do nothing (all values are set to be zeros). If we want to detect edges on all three color channels, juste set all three layers of the filter equally to a edges detector.

Figure ..: Three layers of the filter can be configured differently to detect features of only one, two or all three channels of the input image.

Many filters can be used to detect different kinds of features (vertical, horizontal or 45o edges for example) of input images at the same time. Each filter outputs an 4×44\times44×4 matrix, we can stack them together and end up with a 4×4×24\times4\times24×4×2 output volumne.

The convolution with many filters presented above can be turned into an one-layer convolutional network by adding each 4×44\times44×4 output matrix with its bias - a real number and pass them throuhg an non-linear activation function, such as ReLUReLUReLU. How many parameters are there in one convolutional layer containing 10 filters of 3×3×33\times3\times33×3×3?

Each 3×3×33\times3\times33×3×3 filter has 272727 parameters with one bias bbb. We have totally 28×10=28028 \times 10 = 28028×10=280 parameters to learn for all 101010 filters. Three types of layers in a CNN:

  • Convolution (CONV)
  • Pooling (POOL)
  • Fully connected (FC)

Pooling layers

ConvNets often use pooling layers to reduce the size of the presentation, speed the computation as well as make some of the features that detects a bit more robust. Several pooling layers exist: Max pooling, Average pooling, etc.

Max pooling

The reason behind max pooling is that, if some features are detected anywhere in region (covered by the filter), then keep the highest number. However, the real underling reason explaining why max pooling work well in experiments is unknown.

Figure ..: Example of max pooling. The filter of size 2x2 slides over the input matrix two pixels each step (s = 2) and devide it into different regions. Each output is the max from the corresponding shaped region.

  • There are two hyperparameters (size of the filter fff and strike sss) however, there is NO parameters to learn in a max pooling layer. When fff and sss are fixed, the computation is fixed and gradient descent doesn't change anything.
  • The size of output remains: ⌊n+2p−fs+1⌋\left \lfloor \frac{n + 2p -f}{s} + 1 \right \rfloor⌊sn+2p−f​+1⌋
  • For input volumne, the computation is performed on each of the channels independently.
Average pooling

Instead of taking the maximun value, the average pooling take the average of all values in the shaped region. Max pooling is use much more often than average pooling with one exception which is sometimes very deep in a NN, we use average pooling to collapse the presentation from 7×7×10007\times7\times10007×7×1000 to 1×1×10001\times1\times10001×1×1000 (?).

Why Convolutions?

Two main advantages of convolutional layers over fully connected layers are: Parameter sharing and Sparsity of connections.
If we have a 32×32×332\times32\times332×32×3 dimensional image and use 666 filters of 5×55\times55×5, we end up with 28×28×628\times28\times628×28×6 dimensional output. The input has 307230723072 (=32×32×3=32\times32\times3=32×32×3) units and the output has 470447044704 units. If we use fully connected layers, the weight matrix has size of 3072×47043072\times47043072×4704, so there are almost 141414M parameters. In meanwhile, the convolutional layer has only 6×(25+1)=1566\times (25+1) = 1566×(25+1)=156 parameters. The reason that the ConvNet run through these small parameters is really two reasons: "Parameter sharing" and "Sparsity of connections".

  • Parameter sharing: A feature detector (such as vertical edge detector) that's useful in one part of the image is probably useful in another part of the image. The input is covered by regions and these regions are overlapped. This approach reduces the number of parameters and the parameters are shared among regions. Each of outputs can use the same parameters in lots of positions in the input image. We can see the parameter sharing from feature detector's view, where parameters are used (and reused) to compute all outputs corresponding diffrent parts of the input. If a set of parameters can detect edges at upper lef-hand corner of the image, the same parameters seem probably useful for edges detection at lower right hand corner., so we dont need to learn separate feature detector for two diffrent region of the input image.
  • The ConvNet has also Sparse connection: An output unit is based only on the feature detector and a small resigon of the input image instead of the whole input picture . This is different with the fully connected where an output unit depends on all units of the input image.

An image shifted a few pixels should result in pretty similar features and should probably be assigned the same label.

How to learn parameters?

The parameters learning can be done in two steps:

  • Any setting of the parameters allows us to compute the cost function: J=1m∑i=1mL(y^(i),y(i))J = \frac{1}{m} \sum_{i=1}^m L(\hat{y}^{(i)}, y^{(i)})J=m1​∑i=1m​L(y^​(i),y(i)).
  • Then we use the *gradient descent (or others like momentum) to optimize parameters to reduce JJJ.
